# 方上取送货流程-物理按钮对接说明

## 1. 文档目的

本文档用于指导车间物理按钮与 AGV 后端系统对接。  
本场景仅涉及两个业务动作：

- 按钮A：方上取货（Load）
- 按钮B：方上送货（Unload）

物理按钮侧只需调用对应的两个接口即可，不需要自行拆分 Robokit 底层分段动作。

---

## 2. 对接范围与原则

- 物理按钮系统只负责触发 HTTP 接口。
- 流程编排（抢占控制权、分段导航、等待任务完成）由后端接口内部处理。
- 按钮系统必须做防抖、串行互斥、失败告警。

---

## 4. 请求与响应规范

## 4.1 通用请求头

- `Content-Type: application/json`

## 4.2 方上取货（Load）

### 请求体

```json
{
  "pickup_point": "AP0",
  "nick_name": "operator",
  "timeout_sec": 300,
  "poll_ms": 500,
  "target_height": 0.25,
  "recognize": true,
  "recfile": "plt/p2.plt"
}
```

### 字段说明

- `pickup_point`：必填，取货点站点 ID（如 `AP0`）
- `nick_name`：可选，抢占控制权名称，默认 `operator`
- `timeout_sec`：可选，单段等待超时秒数，默认 `300`
- `poll_ms`：可选，状态轮询间隔毫秒，默认 `500`
- `target_height`：可选，仅取货流程首段 ForkLoad 使用，默认 `0.25`
- `recognize`：可选，默认 `true`
- `recfile`：可选，默认 `plt/p2.plt`

### 成功判定

- HTTP 状态码为 `200`
- 响应 JSON 中 `success = true`

---

## 4.3 方上送货（Unload）

### 请求体

```json
{
  "delivery_point": "AP0",
  "nick_name": "operator",
  "timeout_sec": 300,
  "poll_ms": 500
}
```

### 字段说明

- `delivery_point`：必填，送货点站点 ID（如 `AP0`）
- `nick_name`：可选，默认 `operator`
- `timeout_sec`：可选，默认 `300`
- `poll_ms`：可选，默认 `500`

### 成功判定

- HTTP 状态码为 `200`
- 响应 JSON 中 `success = true`
