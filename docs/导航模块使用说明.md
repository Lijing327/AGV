# AGV Web 控制台 —「导航」模块使用说明

本文说明前端页面 **Robokit 面板 → 分组「导航」** 中各功能的用途、推荐操作顺序与注意事项。底层调用的是 Robokit 导航端口（通常为 `19206`）对应 API，后端路径前缀为：`/api/robokit/navigation/*`。

---

## 1. 使用前提

1. **先连接机器人**  
   在面板顶部填写 IP、端口，点击「连接机器人」，确认状态为「已连接」。

2. **建议先抢占控制权**  
   在 **「控制」** 分组中点击 **「抢占控制」**。若未抢占，部分导航接口可能返回调度/权限类错误（如与控制权相关的 ret_code）。

3. **地图与站点**  
   路径规划、3051、3066 均依赖 **当前机器人地图中的站点 ID** 及 **边（相邻站点须有直接连线）**。站点名需与地图一致（如 `LM1`、`AP1`、`SELF_POSITION` 等）。

4. **自动模式**  
   部分「导航到点」类能力要求小车处于自动/导航模式；若固件已弃用模式切换 API，抢占控制后可直接尝试下发导航。

5. **操作日志**  
   执行结果与错误会以简短文案写入页面底部的 **「操作日志」**，调试时请同时查看日志与浏览器 **Network** 中对应接口响应。

---

## 2. 界面功能总览（从上到下）

| 区块 | 主要作用 | 典型 API |
|------|----------|----------|
| 智能路径规划 | 用 BFS 在已加载地图上生成站点序列，自动填入下方「指定路径」多段表单 | 后端路径规划 + 3066 表单 |
| 一键搬运 | 取货点 ForkLoad → 按规划走路径 → 终点 ForkUnload（模式 A 一次下发 / 模式 B 等 DI 后分段下发） | 3066 |

| 转动 / 托盘旋转 | 固定角度旋转 | 3056 / 3057 |

## 3. 智能路径规划

**用途**：避免手工逐段填写中间站点。输入 **起点**、**终点** 后点击 **「规划路径」**，系统根据地图做最短路径（BFS），并把结果写入下方 **「指定路径 (3066) — 多段编辑」**。

**可选参数**：

- **货叉默认高度/前移**（`start_height`、`fork_mid_height`、`end_height`、`fork_dist`，单位 **米**）：规划成功后可自动带入各段；未填的字段不会下发。
- **规划后末段自动 ForkUnload**：勾选后最后一段会带卸载逻辑，且通常会约束 **`end_height = 0`**（到点卸货）。

**3066 请求体预览**：

- 规划或修改多段后，可点 **「刷新预览」** 查看即将 POST 的 JSON；**「复制 JSON」** 便于对照固件文档或抓包。

**校验**：

- 若多段首尾站点不衔接，预览区会给出 **连续性警告**。
- 若某段缺 `source_id` / `id` / `task_id`，预览会提示错误。

**下一步**：在「指定路径」中确认或微调各段后，点击 **「指定路径(3066)」** 下发。

---

## 4. 一键搬运

**用途**：快速验证「取货 → 行走 → 放货」流程，内部仍走 **3066**。

**填写项**：

- **取货点 / 放货点**：地图站点 ID。
- **模式**  
  - **A**：整段 `move_task_list` **一次下发**（含 ForkLoad 与 ForkUnload）。  
  - **B**：先只下发 **ForkLoad 段**，轮询 **DI** 直到 `status === true`，再下发 **剩余路径 + ForkUnload**。  
    - 需填写 **叉好信号 DI (id)**、**超时 (秒)**、**轮询 (ms)**（与「监控」里 I/O 的 DI id 一致）。
- **货叉参数**：可选，作用于 ForkLoad 段及各导航段的默认高度/前移。

**按钮**：

- **刷新预览**：根据当前取/放货点拉路径规划并更新 JSON（需两点均填写）。
- **执行一键搬运**：按所选模式下发。

**注意**：模式 A 依赖固件在 ForkLoad 内部完成后再走后续段；若现场需要物理到位确认，优先用模式 B。

---

## 5. 指定路径导航（3066）——推荐主功能

**用途**：生产/调试用的 **多段路径**，每段对应地图上一跳。

**每段必填**：

- `source_id`：本段起点站点（或 `SELF_POSITION` 表示从当前位姿出发）。
- `id`：本段终点站点。
- `task_id`：本段任务号，**在整条链中建议唯一**。

**可选**：

- `operation`：`Wait`、`ForkLoad`、`ForkUnload`、`ForkHeight`、`ForkForward` 等。
- 货叉高度、前移：`start_height`、`fork_mid_height`、`end_height`、`fork_dist`（**米**）。

**规则**：

- **相邻段必须首尾相连**：上一段的 `id` 等于下一段的 `source_id`。
- 若只填了高度等而未选 `operation`，系统可能自动补 `ForkHeight`（预览 JSON 中会体现）。

**下发前先等 DI**：勾选后，会先轮询指定 DI，到位后再调用 3066，适合叉齿、挡料等联锁。

**按钮**：

- **+ 添加一段**：增加表单行。
- **指定路径(3066)**：提交当前多段列表。

**常见错误 52702（路径规划失败）**：

- 两站点之间 **没有直接边**；
- 单位错误或数值异常；
- 站点 ID 与地图不一致。

---

## 6. 路径导航（3051）

**用途**：**仅建议单车/测试**。给定 **起点 source_id**、**终点 id**（及可选 `task_id`），走 **单条固定路径**。

**起点**：可填具体站点或 **`SELF_POSITION`**（小车不在命名起点时）。

**出发模式**：

- **直接出发**：立即发 3051。
- **等待 DI 叉好后再出发**：与 3066 类似，先轮询 DI 再发 3051。

**可选**：勾选 **本段附带 ForkUnload**，并填写相应货叉高度参数，用于到点卸货测试。

**失败排查**：与界面红字提示一致——检查地图直连、`SELF_POSITION` 用法、`task_id` 唯一性等。

---

## 7. 导航控制

| 按钮 | 说明 |
|------|------|
| 停止导航 (3052) | 停止当前导航任务 |
| 暂停 (3001) | 暂停当前导航 |
| 继续 (3002) | 继续已暂停的导航 |
| 取消 (3003) | 取消当前导航 |

用于任务执行中的干预；具体行为以固件为准。

---

## 8. 转动 (3056) / 托盘旋转 (3057)

- **3056**：按 **弧度 (rad)** 与可选 **角速度**、**定位模式**（里程/激光）旋转。
- **3057**：托盘旋转，同样以 **rad** 为主。

用于位姿微调或小范围转向，注意周围安全。

---

## 9. 圆弧 (3058) / 启用禁用线路 (3059)

在 **「请求体 JSON」** 中填写固件要求字段后点击对应按钮。示例（圆弧）：`{"radius": 1, "angle": 1.57}`。完整字段请以 Robokit 接口文档为准。

---

## 10. 清除导航路径

- **清除指定路径 (3067)**：按接口定义清除。
- **按 task_id 清除 (3068)**：在输入框填写 `task_id` 后点击。

用于清理残留任务或重试前复位。

---

## 11. 任务链 (3101 / 3115 / 3106)

- **查询任务链状态 (3101)**、**查询所有任务链 (3115)**：结果展示在下方 JSON 区域。
- **执行预存任务链 (3106)**：填写 **预存任务链名称** 后执行。

适合已在线下好任务链名称的标定场景。

---

## 12. 获取路径导航的路径 (3053)

点击 **「获取当前路径」** 查看当前路径相关数据（与 3051 路径导航配套使用时的调试手段）。

---

## 13. 导航状态

显示 **导航状态**、**定位状态** 等摘要；可点击刷新图标从机器人拉取最新（需在已连接前提下）。

---

## 14. 与后端 API 的对应关系（查阅用）

前端通过 `frontend/src/api.js` 调用下列路径（均为 `POST` 或文档注明的 `GET`），详情见 `app/routers/robokit_router.py`：

- `POST /api/robokit/navigation/path` — 3051  
- `POST /api/robokit/navigation/specified-path` — 3066  
- `POST /api/robokit/navigation/stop` — 3052  
- `POST /api/robokit/navigation/pause` — 3001  
- `POST /api/robokit/navigation/resume` — 3002  
- `POST /api/robokit/navigation/cancel` — 3003  
- `POST /api/robokit/navigation/target-path` — 3053  
- `POST /api/robokit/navigation/turn` — 3056  
- `POST /api/robokit/navigation/spin` — 3057  
- `POST /api/robokit/navigation/circular` — 3058  
- `POST /api/robokit/navigation/path-enable` — 3059  
- `POST /api/robokit/navigation/clear-target-list` — 3067  
- `POST /api/robokit/navigation/clear-by-task-id` — 3068  
- `GET /api/robokit/navigation/tasklist-status` — 3101  
- `GET /api/robokit/navigation/tasklist-list` — 3115  
- `POST /api/robokit/navigation/tasklist-execute` — 3106  
- `POST /api/robokit/navigation/move-to` — 导航到点类（参数见路由定义）

路径规划所需地图数据来自当前后端已加载的地图（如通过地图导入/默认地图等），与「智能路径规划」按钮调用的后端逻辑一致。

---

## 15. 建议的首次上手顺序

1. 连接机器人 → **控制** 里 **抢占控制**。  
2. 打开 **导航** → **智能路径规划** 输入起点终点 → **规划路径**。  
3. 检查 **3066 请求体预览** 与连续性提示。  
4. 点击 **指定路径(3066)** 下发。  
5. 用 **导航状态** 与 **操作日志** 观察结果；异常时用 **停止导航** 或 **清除路径** 复位。

如需带货叉与 DI 联锁，在 3066 区块勾选 **下发前先等待 DI**，或在 **一键搬运** 中使用模式 B。
