# AGV 地图模拟系统使用指南

本指南介绍如何使用增强版的 AGV 地图模拟系统，该系统支持导入不同类型的地图文件，并在导入的地图上模拟小车执行任务。

## 功能特点

- 支持导入 `.smap` 格式地图文件，包括贝塞尔曲线路径
- 智能 AGV 初始位置选择，适应不同类型的地图
- 增强的前端渲染，支持贝塞尔曲线和不同类型的节点显示
- AGV 移动路径可视化，显示方向指示器
- 完整的任务执行流程模拟

## 快速开始

### 1. 启动后端服务

```bash
# 在项目根目录下
pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 2. 启动前端服务

```bash
# 在新终端中
cd frontend
npm install
npm run dev
```

### 3. 访问系统

打开浏览器访问 `http://localhost:3000`，即可看到 AGV 调度系统界面。

## 使用流程

### 1. 导入地图

1. 点击界面右侧的"导入地图"按钮
2. 选择一个 `.smap` 或 `.json` 格式的地图文件
3. 系统会自动解析并显示地图
4. 地图信息会在"地图数据"部分显示

### 2. 启动模拟

1. 点击"启动模拟"按钮开始模拟
2. 系统会每秒执行一次"tick"，更新小车位置和任务状态
3. 小车状态和位置会实时更新在界面上

### 3. 创建任务

1. 在"创建任务"部分输入起点和终点节点ID
   - 起点：例如 "R1"
   - 终点：例如 "D1"
2. 点击"创建任务"按钮
3. 系统会自动选择最近的空闲小车分配任务
4. 小车会开始沿规划路径移动

### 4. 监控执行

- 在"车队状态"部分查看所有小车的实时状态
- 在"任务列表"部分查看所有任务的状态
- 在"日志"部分查看系统操作的详细日志

### 5. 停止模拟

点击"停止模拟"按钮暂停模拟，小车将停止移动。

## 地图格式支持

### .smap 格式

系统支持完整的 `.smap` 格式，包括：

- `advancedPointList`: 节点定义
- `advancedCurveList`: 路径定义，包括贝塞尔曲线
- `normalPosList`: 简单点列表（兼容性）

节点类型包括：
- `Station`: 接收区（紫色）
- `Conveyor`: 传送带（蓝色）
- `Delivery`: 配送区（橙色）
- `Assembly`: 装配区（绿色）

路径类型包括：
- `NormalPath`: 直线路径
- `BezierPath`: 贝塞尔曲线路径

### .json 格式

系统也支持标准 JSON 格式，包含 `nodes` 和 `edges` 数组。

## 可视化特性

- 不同类型节点使用不同颜色显示
- 贝塞尔曲线路径以平滑曲线显示
- 小车显示为圆形，内有编号
- 移动中的小车显示方向指示器
- 路段占用状态实时更新

## 测试模拟

项目包含一个测试脚本 `test_simulation_flow.py`，可用于验证系统功能：

```bash
python test_simulation_flow.py
```

该脚本会：
1. 导入测试地图
2. 初始化小车
3. 创建测试任务
4. 启动并监控模拟
5. 报告测试结果

## 故障排除

### 地图导入失败

1. 检查地图文件格式是否正确
2. 确保节点和边的引用关系正确
3. 查看浏览器控制台的错误信息

### 小车不移动

1. 确保已点击"启动模拟"按钮
2. 检查起点和终点是否存在于地图中
3. 查看日志中的错误信息

### 路径显示异常

1. 确保地图中节点坐标正确
2. 检查贝塞尔曲线控制点是否合理
3. 刷新页面重试

## 技术架构

- 后端：FastAPI + Python
- 前端：Vue 3 + Vite
- 地图渲染：Canvas 2D
- 路径规划：BFS 最短路径算法
- 模拟引擎：基于 tick 的异步事件循环
